# 审批模型

状态：草案。

审批模型用于保护工作区，避免未审阅的写入、命令执行、网络访问和破坏性操作。

## 风险等级

| 等级 | 示例 | 默认行为 |
| --- | --- | --- |
| Read | read file、search、git status | 允许 |
| Write | apply patch、格式化已跟踪文件 | 询问 |
| Exec | 测试、构建、lint 命令 | 询问 |
| Network | 下载依赖、远程 API | 询问 |
| Destructive | 删除、reset、清理未跟踪文件 | 总是询问 |

## 审批事件

审批请求应展示：

- 请求的工具
- 风险等级
- 工作目录
- 精确命令或文件路径
- 预期影响
- 安全时可展示持久批准选项

## 拒绝审批

审批被拒绝时，Agent Core 记录拒绝结果，并选择明确下一步：

- 请求用户选择其他路径
- 继续只读工作
- 停止 run

Agent Core 不得用相似命令绕过已拒绝审批。
